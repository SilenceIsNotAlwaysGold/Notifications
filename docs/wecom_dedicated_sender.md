# 企业微信专用通知账号接入

本文说明如何用独立企业微信成员承载外部法务群的自动提醒。该通道默认关闭，不替代官方会话内容存档。发送网关可以使用第三方服务，也可以使用项目内置的自托管 Android RPA 网关；新部署优先采用自托管方案。

## 1. 创建专用成员

建议成员名称使用“致和法务通知助手”，头像和对外名称明确表示其通知用途。

- 不授予超级管理员权限。
- 只开放客户联系和必要客户群权限。
- 只加入需要自动提醒的法务外部群。
- 如需自动修改真实群名，将该账号设置为群主或群管理员；仅发送消息时无需扩大管理权限。
- 建议将该成员纳入会话内容存档范围，确保其所在目标群可以被官方归档链路发现。

## 2. 发送网关 POC

自托管方案参见 [企业微信外部群自托管发送方案](./wecom_self_hosted_sender.md)。先使用测试成员和测试外部群，准备：

- `WECOMAPI_BASE_URL`
- `WECOMAPI_TOKEN`
- `WECOMAPI_GUID`
- 测试群对应的目标白名单 ID

不要在 POC 中发送真实案号、判决书、身份证明、收款信息或客户隐私。

## 3. 配置服务

```env
WECOM_SEND_MODE=wecomapi
WECOMAPI_BASE_URL=http://127.0.0.1:8092
WECOMAPI_API_PATH=/wecom/finder/api
WECOMAPI_TOKEN=自托管网关Token
WECOMAPI_GUID=自托管网关robot_id
WECOMAPI_MIN_INTERVAL_SECONDS=3
WECOMAPI_DAILY_LIMIT=200
WECOMAPI_FAILURE_THRESHOLD=3
WECOMAPI_COOLDOWN_SECONDS=300
```

运行配置预检：

```bash
python -m app.cli check-config
```

预检会明确提示当前使用的是非官方发送通道。Token 不应写入 Git、前端代码或发送日志。

## 4. 建立群 ID 映射

1. 打开管理端“归档群”。
2. 通过官方会话内容存档发现目标群，确认官方 `roomid` 和显示名称。
3. 在 `WECOM_SENDER_TARGETS_JSON` 为该群创建稳定白名单 ID，例如 `zhihe-legal`。
4. 将该白名单 ID 填入该群的“发送目标 ID”并保存。
5. 先发送无敏感内容的测试提醒，人工确认目标群正确。

系统不会把官方 `roomid` 猜成发送目标。缺少显式映射时，提醒保持失败/重试状态且不会请求发送网关。

## 5. 运行边界

- 当前协议模式只发送文本，不保证真正 `@` 外部成员。
- 不通过发送网关上传或下载法务文件。
- 单账号发送会串行执行，并受最小间隔和每日上限约束。
- 连续失败达到阈值后自动熔断，冷却期内不再请求网关。
- 发送日志记录业务群、内容摘要、状态码和错误，不记录 Token。
- 紧急停用时将 `WECOM_SEND_MODE` 切换为 `mock` 并重启服务。

## 6. 上线门槛

- 完成测试账号灰度，验证至少 7 天稳定性。
- 自托管网关与手机端均由我方控制，不依赖第三方会话凭证托管。
- 客户书面确认非官方 UI 自动化风险和停用预案。
- 对每个真实群逐一核对 ID 映射，不允许批量猜测或按群名自动匹配。
