# 企业微信接入可行性说明

本文用于第十六阶段 POC：验证企业微信外部法务群是否可以被系统可靠接入。

## 1. 企业微信群机器人 Webhook

企业微信群机器人 webhook 只能作为发送消息出口。

- 可以用于向群内发送文本、markdown 等机器人消息。
- 不能读取群消息。
- 不能作为群消息监听方案。
- 对外部群是否可用，需要客户在目标外部群中实际验证。
- 如果目标外部群不能添加机器人，则 webhook 方案不可用于该群提醒。

因此，webhook POC 的验证目标是：目标外部群是否能添加机器人，以及系统通过 webhook 发送的测试消息是否能被群成员看到。

## 2. 企业微信会话内容存档

企业微信会话内容存档是读取群消息的官方路线。

真实接入前需要客户确认并提供：

- 企业 ID `corp_id`
- 会话内容存档 `secret`
- private key 或 private key 文件
- `public_key_ver`
- 企业微信后台已开通会话内容存档权限

还必须进一步确认：

- 会话内容存档是否覆盖外部群。
- 参与外部群聊的客户员工是否在存档范围内。
- 是否可以拉取图片、文件、PDF 等媒体。
- 会话内容存档 SDK / API 是否允许当前业务所需的消息类型。

如果客户没有会话内容存档权限，系统无法通过官方方式读取企业微信外部群消息。

## 3. 不建议方案

不建议使用以下方案读取企业微信群消息：

- 个人号 hook
- 模拟网页版
- 抓 cookie
- 非官方客户端监听
- 逆向协议或非官方注入方式

原因：

- 稳定性差，容易因企业微信升级失效。
- 合规风险高。
- 容易触发封控。
- 难以向客户安全、审计和法务部门解释。
- 不适合处理判决书、付款截图等敏感材料。

## 4. 当前系统状态

当前系统已经具备以下工程基础：

- 已有企业微信群机器人 webhook 发送 adapter。
- 已有企业微信会话内容存档 adapter 骨架。
- 已有 `/api/v1/legal/wecom-archive/replay` 回放接口，可模拟归档消息进入业务链路。
- 已有媒体文件、OCR、提醒、腾讯文档 mock 同步等后续处理链路。

但需要明确：

- 当前尚未验证真实企业微信外部群消息读取。
- 当前尚未验证客户是否具备会话内容存档权限。
- 当前尚未验证目标外部群是否允许添加群机器人。
- replay 模拟成功不代表已经真实接入企业微信生产环境。

## 5. POC 建议顺序

1. 在目标外部群尝试添加企业微信群机器人。
2. 使用 `/api/v1/legal/wecom-poc/send-test` 发送测试消息。
3. 确认群成员是否收到消息。
4. 与客户确认是否已开通会话内容存档。
5. 使用 `/api/v1/legal/wecom-poc/archive-check` 检查配置字段是否齐全。
6. 确认会话内容存档是否覆盖目标外部群和群内员工。
7. 再进入真实会话内容存档 SDK / API 联调。
